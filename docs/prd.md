# 需求文档

## 1. 应用概述

### 1.1 应用名称
OwlOS v1.0

### 1.2 应用描述
OwlOS是一个意图驱动的Agent运行时系统，用户通过单一对话入口与系统交互，系统内置Orchestrator自动判断任务复杂度，在单聊（Single）、群聊（Squad）、自动化（Auto）三种模式间无缝切换，提供智能化的任务执行与协作能力。

### 1.3 设计系统规范

**Aurora设计系统**
- 深色模式：
  + 深空背景：#020617到#042f2e的径向渐变
  + 极光强调色：#00f2c3（极光青）→#38bdf8（冰蓝）→#7c3aed（电光紫）的线性渐变
  + 玻璃拟态：rgba(255,255,255,0.03)到rgba(255,255,255,0.08)渐变背景，backdrop-blur 16-24px
- 浅色模式：
  + 浅灰背景：#f8fafc到#f1f5f9的径向渐变
  + 深色文字：#0f172a（主文字）、#475569（次要文字）
  + 极光强调色：保持#00f2c3→#38bdf8→#7c3aed渐变
  + 玻璃拟态：rgba(255,255,255,0.6)到rgba(255,255,255,0.8)渐变背景，backdrop-blur 16-24px
- 字体：Inter, SF Pro Display, system-ui
- 主题切换：通过Tailwind dark:前缀实现，基于class-based模式
- 响应式：桌面端优先

**技术栈**
- React 18 + Tailwind CSS + Lucide React（图标）
- Framer Motion：所有面板滑入滑出、消息进入、布局切换动画
- Recharts：计费图表、Token趋势图

## 2. 用户与使用场景

### 2.1 目标用户
- 需要智能化任务处理的个人用户
- 需要团队协作与自动化工作流的企业用户
- 需要集成AI能力的开发者

### 2.2 核心使用场景
- 通过对话完成单一任务（单聊模式）
- 复杂任务自动升级为多Agent协作（群聊模式）
- 定期执行自动化工作流（自动化模式）
- 管理知识库文档并进行检索测试
- 浏览和使用工具市场中的工具与模板
- 配置Agent、团队、工作流及系统设置
- 在深色与浅色主题间切换以适应不同使用环境
- 配置知识库切片规则
- 通过AI对话方式检索知识库内容
- 编辑知识库文档与知识块

### 3.2 全局架构

#### 3.2.1 左侧边栏
- 支持64px图标模式与240px展开模式切换
- 展示核心模块入口：对话、知识库、工具市场、设置
- 当前选中模块高亮显示

#### 3.2.2 顶部全局栏
- 高度56px
- 左侧：面包屑导航，展示当前位置层级
- 中间：全局搜索框
- 右侧：主题切换按钮（太阳/月亮图标）、用户头像、通知图标、快捷工具入口
- 主题切换按钮功能：
  + 深色模式时显示太阳图标，点击切换到浅色模式
  + 浅色模式时显示月亮图标，点击切换到深色模式
  + 切换时触发AppContext中的darkMode状态更新

#### 3.2.3 主内容区
- 根据当前选中模块动态展示对应内容
- 支持Framer Motion动画过渡

### 3.3 对话模块

#### 3.3.1 空状态
- 首次进入或无会话时展示
- 显示欢迎信息与快速开始引导
- 提供示例问题供用户点击

#### 3.3.2 会话列表
- 展示至少3个会话（单聊、群聊、自动化各一个）
- 每个会话显示标题、最后消息预览、时间戳
- 支持点击切换当前会话
- 支持新建会话、删除会话

#### 3.3.3 消息流容器
- 展示当前会话的8条消息
- 消息类型包括：
  - 用户消息
  - Agent回复消息
  - 工具调用消息
  - 思维链展示
  - 代码块展示
  - 群聊@提及消息
- 消息进入时使用Framer Motion动画

#### 3.3.4 自动升级交互
- 当任务复杂度超过阈值时，系统自动提示升级为群聊或自动化模式
- 展示升级提示卡片，用户可确认或取消
- 确认后自动切换到对应模式

#### 3.3.5 底部输入区
- 多行文本输入框
- 支持快捷键发送（Enter）
- 支持上传文件、插入表情、@提及
- 显示输入字符计数

### 3.4 群聊模式

#### 3.4.1 团队面板
- 展示4个Agent：
  - Aria（紫色）
  - Coder（青色）
  - Muse（蓝色）
  - Analyst（琥珀色）
- 每个Agent显示头像、名称、当前状态（在线/离线/工作中）

#### 3.4.2 任务看板
- 支持看板视图与甘特图视图切换
- 看板视图：展示6个任务卡片，分为待办、进行中、已完成三列
- 甘特图视图：展示任务时间线与依赖关系
- 任务卡片包含：任务标题、负责Agent、截止时间、优先级

#### 3.4.3 群聊消息流
- 展示6条群聊消息
- 消息包含Agent头像、名称、消息内容
- 支持@提及特定Agent
- 支持回复、引用消息

### 3.5 自动化模式

#### 3.5.1 执行日志面板
- 展示【每日竞品监控】工作流执行状态
- 显示4个节点：
  - 2个已完成节点（绿色勾选图标）
  - 1个进行中节点（蓝色加载图标）
  - 1个待执行节点（灰色待定图标）

- 支持删除切片规则：

## 4. 业务规则与逻辑

### 4.1 模式自动切换规则
- 系统内置Orchestrator判断任务复杂度
- 单聊模式：适用于简单、单一任务
- 群聊模式：当任务需要多个Agent协作时自动升级
- 自动化模式：当任务需要定期执行或包含多个步骤时自动升级
- 用户可手动确认或取消升级

### 4.2 消息流规则
- 消息按时间顺序展示，最新消息在底部
- 工具调用消息展示调用的工具名称与参数
- 思维链消息展示Agent的推理过程
- 代码块消息支持语法高亮
- 群聊@提及消息高亮显示被提及的Agent

### 4.3 任务看板规则
- 任务卡片可在待办、进行中、已完成三列间拖拽移动
- 任务状态变更时自动更新负责Agent的工作状态
- 甘特图视图根据任务时间线自动排列

### 4.4 工作流执行规则
- 工作流按节点顺序依次执行
- 节点执行完成后自动触发下一节点
- 节点执行失败时暂停工作流，记录错误日志
- 支持手动重试失败节点

### 4.5 知识库检索规则
- AI对话式检索时，系统根据用户输入问题匹配知识块
- 匹配结果按相似度得分排序
- AI回复中展示知识块内容、来源文档、相似度得分
- 支持多轮对话，用户可追问或补充问题

### 4.6 知识库编辑规则
- 点击文档列表中的文档后，右侧展示该文档的所有知识块
- 知识块按索引顺序展示
- 删除文档时，该文档的所有知识块同步删除
- 上传新文档后，系统自动处理并生成知识块

### 4.7 切片规则管理规则
- 新建切片规则时，名称不可重复
- 策略类型为固定长度时，必须填写块大小与重叠量
- 策略类型为按句子、按段落、语义分割时，块大小与重叠量可选填
- 预处理规则为多选字段，可同时选择多个选项
- 删除切片规则前需二次确认，避免误删

### 4.8 计费规则
- 按Token使用量计费
- 每日统计Token使用量并更新趋势图
- 月末生成账单并展示总消费金额

### 4.9 可视化节点画布操作规则
- 节点拖拽：鼠标按下节点后移动鼠标，节点跟随鼠标移动，松开鼠标后节点固定在新位置
- 连线创建：鼠标按下节点输出端点后拖拽到目标节点输入端点，松开鼠标后创建连线
- 连线删除：点击连线后连线高亮显示，按删除键或点击删除按钮删除连线
- 节点删除：点击节点上的删除按钮后删除节点，同时删除与该节点相关的所有连线
- 画布缩放：鼠标滚轮向上滚动放大画布，向下滚动缩小画布
- 画布平移：鼠标按下画布空白区域后拖拽，画布跟随鼠标移动
- 节点配置：点击节点后右侧展示配置面板，修改配置后点击保存按钮保存配置
- 节点类型约束：输入节点只能有输出端点，输出节点只能有输入端点，其他节点同时具有输入和输出端点
- 连线方向：连线只能从输出端点连接到输入端点，不允许反向连接
- **自动布局规则**：
  - 点击顶部工具栏「自动布局」按钮后，系统执行有向图层次化布局算法
  - 算法按拓扑层次从左到右排列节点，同一层次的节点垂直居中对齐
  - 节点尺寸为168×72px，层次间水平间距为200px，同层节点垂直间距为100px
  - 布局切换时，节点从当前位置平滑移动到新位置，动画时长为600ms
  - 布局完成后，画布自动缩放与平移，确保所有节点可见（fit to view）

### 4.10 主题切换规则
- AppContext中维护darkMode状态（布尔值）
- darkMode为true时，在html元素上添加「dark」class；为false时移除「dark」class
- 顶部全局栏主题切换按钮点击时，触发AppContext中的darkMode状态切换
- 外观主题设置页的深色/浅色切换开关点击时，同样触发AppContext中的darkMode状态切换
- 主题切换时，所有页面与组件根据Tailwind dark:前缀自动应用对应样式
- index.css中定义完整的light/dark CSS变量，供全局样式使用

### 4.11 知识库表单规则
- 新建知识库时，必须填写知识库名称与向量库地址
- 向量库地址格式示例：http://localhost:6333 或远程 Qdrant/Milvus 地址
- 编辑知识库时，可修改知识库名称、向量库地址、描述

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 网络连接失败 | 显示错误提示，支持重试 |
| 消息发送失败 | 显示发送失败标识，支持重新发送 |
| 文档上传失败 | 显示错误状态，提示用户检查文件格式或大小 |
| 工作流执行失败 | 暂停工作流，记录错误日志，支持手动重试 |
| 知识库检索无结果 | AI回复「未找到相关内容」 |
| API密钥过期 | 显示过期提示，引导用户重新生成 |
| 计费余额不足 | 显示余额不足提示，引导用户充值 |
| 会话列表为空 | 显示空状态，引导用户新建会话 |
| 工具市场加载失败 | 显示加载失败提示，支持刷新 |
| 节点连线形成循环 | 显示警告提示，阻止创建循环连线 |
| 节点拖拽超出画布边界 | 自动调整画布视图，确保节点可见 |
| 节点配置未保存 | 关闭配置面板前提示用户保存或放弃修改 |
| 自动布局时画布无节点 | 显示提示「画布中无节点，无法执行自动布局」 |
| 自动布局时节点形成循环依赖 | 显示警告提示「检测到循环依赖，无法执行自动布局」 |
| 切片规则名称重复 | 显示错误提示「规则名称已存在，请使用其他名称」 |
| 删除切片规则时取消确认 | 关闭确认弹窗，不执行删除操作 |
| 知识库编辑页面无文档 | 显示空状态，引导用户上传文档 |
| 知识库编辑页面文档处理中 | 显示处理中状态，禁止删除或编辑 |
| 向量库地址格式错误 | 显示错误提示「向量库地址格式不正确，请检查后重新输入」 |
| 切片规则预处理规则未选择 | 允许保存，预处理规则为空 |

## 6. 验收标准

1. 用户进入OwlOS，默认展示对话模块空状态
2. 用户在底部输入区输入问题并发送，系统返回Agent回复消息
3. 当任务复杂度超过阈值时，系统自动提示升级为群聊模式，用户确认后切换到群聊模式
4. 在群聊模式下，用户查看团队面板中的4个Agent状态，并在任务看板中查看6个任务卡片
5. 用户切换到自动化模式，查看【每日竞品监控】工作流的执行日志，确认4个节点状态正确显示
6. 用户进入知识库列表页，点击顶部「切片规则管理」图标按钮，切换到切片规则管理页
7. 用户在切片规则管理页点击「新建」按钮，填写名称、策略类型（固定长度）、块大小、重叠量、预处理规则（勾选「自动清洗无效内容」和「过滤低质量短块与乱码内容」）、描述后保存，新规则出现在列表中
8. 用户点击切片规则列表中的「编辑」按钮，修改块大小和预处理规则（新增勾选「保留表格结构化格式，不强行拆分」）后保存，列表中该规则的块大小和预处理规则更新
9. 用户点击切片规则列表中的「删除」按钮，弹出二次确认弹窗，确认后该规则从列表中移除
10. 用户返回知识库列表页，点击「新建知识库」按钮，填写知识库名称、向量库地址（http://localhost:6333）、描述后保存，新知识库出现在列表中
11. 用户点击知识库列表中的「编辑」按钮，修改向量库地址为远程地址后保存，列表中该知识库的向量库地址更新
12. 用户从知识库列表点击「编辑」进入知识库编辑页面，查看顶部统计信息（文档数、知识块数、存储大小、处理中数量），点击左侧文档列表中的1个文档，右侧展示该文档的所有知识块
13. 用户进入知识库检索页面，输入问题后AI返回匹配的知识块内容、来源文档、相似度得分
14. 用户进入工具市场，浏览8个工具卡片，点击其中1个工具查看详情并安装
15. 用户进入设置-工作流编排，从左侧工具栏拖拽1个Agent节点和1个工具节点到画布中，拖拽连线连接两个节点，点击Agent节点后在右侧配置面板中修改配置并保存
16. 用户在工作流编排画布中点击顶部工具栏「自动布局」按钮，节点平滑移动到新位置并自动适配画布视图
17. 用户依次查看专家配置（3个Agent）、团队编排（2个模板）、计费中心（本月消费¥13.52及7天Token趋势图）
18. 用户点击顶部全局栏的主题切换按钮（太阳图标），系统切换到浅色模式，所有页面与组件应用浅色主题样式
19. 用户进入设置-外观主题，点击深色/浅色切换开关，系统切换回深色模式，所有页面与组件应用深色主题样式
